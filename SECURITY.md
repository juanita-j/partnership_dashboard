# 보안 점검 및 개인정보 보호

개인정보가 포함된 파트너사 DB 대시보드의 보안 점검 결과와 권장 사항입니다.

---

## 1. 조치 완료 사항

### 1.1 API 인증 미적용 (치명적) → **해결됨**

- **문제**: `/api/partners`, `/api/export/xlsx`, `/api/import/*` 등 API가 **인증 없이** 호출 가능했음. 외부에서 URL만 알면 개인정보 목록·엑셀 다운로드·수정·삭제가 가능한 상태였음.
- **조치**: 미들웨어에서 **모든 `/api/*` 요청**(단, `/api/auth/*` 제외)에 대해 대시보드 쿠키(`dashboard_auth`) 검사 추가. 비밀번호가 설정된 환경에서는 인증된 사용자만 API 호출 가능.

### 1.2 SQL 인젝션

- **확인 결과**: Prisma ORM 사용, Raw SQL 미사용. **별도 조치 불필요.**

### 1.3 XSS (크로스 사이트 스크립팅)

- **확인 결과**: React 기본 이스케이프, `dangerouslySetInnerHTML` 미사용. **별도 조치 불필요.**

## 2. 추가 권장 사항

### 2.1 HTTPS 필수

- 배포 환경(Vercel 등)에서는 **HTTPS만** 사용하도록 설정. 비밀번호·쿠키가 평문으로 노출되지 않도록 함.

### 2.2 환경 변수 보안

- `DASHBOARD_PASSWORD`, `DASHBOARD_ALLOWED_IDS`, `DATABASE_URL` 등은 **절대 코드/저장소에 넣지 말 것**. Vercel·Railway 등에서 환경 변수로만 관리.

### 2.3 로그인 시도 제한 (선택)

- 현재 로그인 API에 **요청 횟수 제한(rate limit)** 이 없음. 브루트포스 공격에 노출될 수 있음.
- 필요 시 Vercel Edge Config, Upstash Redis, 또는 미들웨어/API 내 단순 카운터로 **IP·계정당 시도 횟수 제한** 적용 권장.

### 2.4 쿠키 설정

- 이미 `httpOnly`, `secure`(production), `sameSite: "lax"` 적용됨. **유지 권장.**

### 2.5 에러 메시지

- API 에러 시 상세 스택/DB 내용을 클라이언트에 반환하지 않도록 되어 있음. **production 빌드**에서도 동일하게 유지할 것.

## 3. 개인정보 취급 요약

| 구분 | 내용 |
|------|------|
| 수집·저장 | 파트너사 이름, 휴대폰, 회사, 부서, 직함, 이메일, 주소, 연도별 DAN/선물 정보 등 |
| 접근 제어 | DASHBOARD_PASSWORD + DASHBOARD_ALLOWED_IDS 설정 시, 해당 ID·비밀번호로 로그인한 사용자만 페이지·API 접근 가능 |
| 전송 | 로그인·API 호출 시 쿠키 사용. 배포 시 HTTPS 사용으로 전송 구간 암호화 권장 |
| 저장 | DB(PostgreSQL 등). DB 접근 권한·백업 구간 암호화 등 인프라 보안 별도 점검 권장 |

---

## 4. 정기 점검 체크리스트

- [ ] 환경 변수에 비밀번호·허용 ID가 코드/저장소에 없음
- [ ] 배포 URL이 HTTPS로만 접속됨
- [ ] 로그인 없이 `/api/partners`, `/api/export/xlsx` 등 직접 호출 시 401 반환
- [ ] DB·백업 접근 권한이 최소 권한으로 제한됨
